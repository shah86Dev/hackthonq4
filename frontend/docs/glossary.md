# Glossary

This glossary contains key terms and concepts used throughout the Physical AI & Humanoid Robotics textbook.

## A

**Action**: A ROS2 communication pattern for long-running tasks that provides feedback and can be canceled.

**Agent**: An entity that perceives its environment and takes actions to achieve goals, often referring to AI agents in this textbook.

**API (Application Programming Interface)**: A set of rules and protocols for building and interacting with software applications.

**Artificial Intelligence (AI)**: The simulation of human intelligence processes by machines, especially computer systems.

## B

**Backend**: The server-side component of a web application that handles data processing and business logic.

**Behavior Tree**: A mathematical model of action sequences used in robotics and AI to create complex behaviors.

**Boolean**: A data type with two possible values: true or false.

## C

**Callback**: A function passed as an argument to another function, which is then executed at a specific time.

**Chatbot**: A computer program designed to simulate conversation with human users, especially over the Internet.

**Chunk**: A segment of content used for Retrieval-Augmented Generation (RAG) systems.

**CI/CD (Continuous Integration/Continuous Deployment)**: Practices for frequently integrating code changes and automatically deploying them.

**Cloud Computing**: The delivery of computing services over the Internet ("the cloud").

## D

**Docusaurus**: A modern static website generator for building documentation websites.

**DDS (Data Distribution Service)**: Middleware protocol and API standard for distributed real-time systems, used in ROS2.

**Debugging**: The process of finding and fixing errors in computer programs.

**Deep Learning**: A subset of machine learning based on artificial neural networks with representation learning.

**Deployment**: The process of making an application available for use.

## E

**Embodied AI**: Artificial intelligence that interacts with the physical world through a body or robotic platform.

**Endpoint**: A remote computing node that provides a communication point in a network.

**Environment**: The physical or virtual space in which a robot operates.

## F

**FastAPI**: A modern, fast web framework for building APIs with Python 3.7+ based on standard Python type hints.

**Feedback**: Information about the results of a process used to modify subsequent actions.

**Framework**: A platform for developing software applications that provides a foundation.

## G

**Gazebo**: A 3D simulation environment for robotics that provides realistic physics and rendering.

**GPU (Graphics Processing Unit)**: A specialized electronic circuit designed to rapidly manipulate and alter memory to accelerate the creation of images in a frame buffer.

**Grounding**: The process of connecting AI responses to specific, reliable sources of information.

## H

**Hardware**: The physical components of a computer or robotic system.

**Humanoid Robot**: A robot with a human-like body structure, typically having a head, torso, two arms, and two legs.

## I

**Isaac Sim**: NVIDIA's robotics simulation application based on NVIDIA Omniverse.

**Integration**: The process of combining different subsystems into a single system.

**Intelligent Agent**: A system that perceives its environment and takes actions to maximize its chance of achieving goals.

## L

**Lab Exercise**: A practical activity that demonstrates concepts learned in theoretical content.

**Launch File**: A ROS2 file that specifies how to start multiple nodes with specific configurations.

**Learning Objective**: A clear statement of what a learner should be able to do after completing a lesson or course.

## M

**Machine Learning**: The study of computer algorithms that can improve automatically through experience.

**Middleware**: Software that provides common services and capabilities to applications beyond what's offered by the operating system.

**Module**: A self-contained component of a larger system, particularly referring to curriculum modules in this textbook.

**Multilingual**: Supporting or using several languages.

## N

**Neural Network**: A series of algorithms that endeavors to recognize underlying relationships in a set of data through a process that mimics how the human brain operates.

**Node**: A process that performs computation in ROS2.

## O

**OpenAI**: An artificial intelligence research laboratory consisting of the for-profit OpenAI LP and the non-profit OpenAI Inc.

**OpenAPI**: A specification for machine-readable interface files for describing, producing, consuming, and visualizing RESTful web services.

## P

**Parameter**: A value that can be changed to configure node behavior in ROS2 without recompiling.

**Personalization**: The process of tailoring content or experiences to individual users based on their preferences or behavior.

**Physics Engine**: Software that simulates the behavior of physical objects using mathematical models.

**Publish-Subscribe Pattern**: A messaging pattern where senders publish messages without knowledge of specific receivers.

## Q

**Qdrant**: An open-source vector similarity search engine with extended filtering support.

**QoS (Quality of Service)**: A set of policies that define the characteristics of data delivery in ROS2.

**Quiz**: A form of assessment that tests knowledge through questions.

## R

**RAG (Retrieval-Augmented Generation)**: A technique that combines information retrieval with text generation to improve the accuracy of language models.

**Real-time System**: A system that must respond to inputs within a specified time limit.

**Repository**: A central location in which data is stored and managed, especially in version control systems.

**Robot Operating System (ROS)**: A flexible framework for writing robot software.

**ROS2**: The second generation of the Robot Operating System.

## S

**Service**: A ROS2 communication pattern for synchronous request-response interactions.

**Simulation**: The imitation of the operation of a real-world process or system over time.

**Static Site Generator**: A software that creates HTML pages from templates or components with content provided in various formats.

**Subagent**: A specialized agent that performs specific tasks as part of a larger AI system.

## T

**Topic**: A named bus over which nodes exchange messages in ROS2.

**Type Hint**: An annotation that specifies the expected type for a variable, function parameter, or return value.

## U

**Urdu**: An Indo-Aryan language spoken in South Asia, used in this textbook as an example of multilingual support.

**User Story**: A description of a feature from the perspective of the end user.

## V

**Vector Database**: A database that stores and indexes vector embeddings for similarity search.

**VLA (Vision-Language-Action) Models**: AI models that can process visual, textual, and action information simultaneously.

**Version Control**: A system that records changes to files over time so that you can recall specific versions later.

## W

**Web API**: An application programming interface for the Web.

**Web-based**: Operated or accessed through a web browser.

## Y

**YAML**: A human-readable data serialization language commonly used for configuration files.

## Z

**Zero-Shot Learning**: The ability of a model to perform a task without having been explicitly trained on examples of that task.
